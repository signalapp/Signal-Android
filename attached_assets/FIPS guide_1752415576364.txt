Below is a hands-on, copy-paste-ready walkthrough you can hand to any new teammate.
It assumes the shallow-clone workflow (no sub-modules) and a clean workstation.

0. Prerequisites (once per machine)
1. Java 17 JDK
Mac: brew install openjdk@17??Ubuntu: sudo apt-get install openjdk-17-jdk
2. Android Studio– install from https://developer.android.com/studio and run it once so it downloads the Android SDK.
3. Git ? 2.30 and rsync (both pre-installed on macOS and most Linux distros).
4. Rust toolchain (for native FIPS bridge)
5. curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
6. rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android
7. cargo install cargo-ndk
8. OpenSSL FIPS provider binaries for Android (see the Compilation Guide PDF). Copy them to
fips-crypto-bridge/libs/openssl/<abi>/libcrypto.so and …/fips.so.

1. Clone the bootstrap repo
git clone git@github.com:<your-org>/fips-signal-android.git
cd fips-signal-android

2. Pull a shallow copy of upstream Signal-Android and merge sources
Tip: adjust the tag when a new Signal release drops.
# Grab only the latest commit of the desired tag
TMP=$(mktemp -d)
git clone --depth=1 --branch v7.38.1 https://github.com/signalapp/Signal-Android "$TMP"

# Copy Java/Kotlin sources & resources into our app module
rsync -a --delete "$TMP"/app/src/ app/src/
rsync -a --delete "$TMP"/app/res/ app/res/

rm -rf "$TMP"

3. Remove duplicate / obsolete files
# Example: keep the MDM-aware StartChatDialogFragment, delete the old one
rm app/src/main/java/org/thoughtcrime/securesms/StartChatDialogFragment.java   # if present
Use git status to verify only expected deletions took place.

4. Verify a local build
./gradlew assembleDebug
Successful output ends with:
BUILD SUCCESSFUL in … seconds
...
> Task :app:packageDebug

5. Commit & push
git add app/src app/res
git commit -m "Pull upstream Signal v7.38.1 (shallow) into app module"
git push origin main

6. CI (GitHub Actions) smoke test
1. Ensure .github/workflows/android.yml exists (see repo).
2. Push triggers a build; check Actions tab ? Build debug APK.
3. Download the app-debug.apk artefact and install on a test device.

7. Open in Android Studio
1. File ? Open… and select the repo root.
2. Let Gradle sync and download any missing SDK platforms.
3. Hit Run ? to deploy to an attached emulator or device.

8. Day-to-day upstream updates
# When Signal v<NEW> is released
TMP=$(mktemp -d)
git clone --depth=1 --branch v<NEW> https://github.com/signalapp/Signal-Android "$TMP"
rsync -a --delete "$TMP"/app/src/ app/src/
rsync -a --delete "$TMP"/app/res/ app/res/
rm -rf "$TMP"

# Fix any merge conflicts or new duplicates, then:
./gradlew assembleDebug   # quick sanity build
git add app/src app/res
git commit -m "Bump upstream Signal to v<NEW>"
git push

Cheatsheet
* Local build: ./gradlew clean assembleDebug
* Run unit tests: ./gradlew testDebugUnitTest
* Update Rust deps: cd fips-crypto-bridge && cargo update
* Generate release build: ./gradlew assembleRelease
Hand these steps to the new hire; they’ll be building FIPS-Signal in under 30 minutes. ??
