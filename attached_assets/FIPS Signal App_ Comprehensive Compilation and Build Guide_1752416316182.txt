FIPS Signal App: Comprehensive Compilation and Build Guide
1.0 Introduction
This document provides a comprehensive guide for compiling the FIPS-compliant Signal-Android application from source. It is intended for developers and build engineers responsible for creating the final application package (APK).
This guide assumes you are working on a Linux-based or macOS development machine and are familiar with standard software development tools and the command line. The process involves multiple stages, including compiling a FIPS-validated cryptographic library from source, setting up the Android project, and running the Gradle build.
2.0 Prerequisites: Development Environment Setup
Before you begin, ensure your development environment has the following tools installed and configured.
? Java Development Kit (JDK): Version 11 or higher is required by the Signal Android project. You can install it via your system's package manager or download it from Oracle or OpenJDK.
# Example on Ubuntu
sudo apt-get install openjdk-11-jdk

? Android Studio: Download and install the latest version of Android Studio. This is the easiest way to get the required Android SDK and build tools.
? Android SDK & NDK:
? Within Android Studio, use the SDK Manager to install the Android SDK Platform for API Level 34.
? From the "SDK Tools" tab, install the Android NDK (Side by side), version 25.1.8937393. This specific version is referenced in our build scripts.
? Ensure the ANDROID_HOME and ANDROID_NDK_HOME environment variables are set correctly in your shell profile (e.g., ~/.bashrc, ~/.zshrc).
export ANDROID_HOME=$HOME/Android/Sdk
export ANDROID_NDK_HOME=$HOME/Android/Sdk/ndk/25.1.8937393
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools

? Rust Toolchain: Install Rust using rustup. This will also install cargo, the Rust package manager.
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"

? Add the necessary Android targets for Rust:
rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android

? CMake: Ensure you have CMake version 3.22.1 or higher installed. You can install it via your system's package manager or from the Android Studio SDK Manager.
? Git: The version control system used to clone the source code.
3.0 Stage 1: Compiling the OpenSSL FIPS Provider
This is the most complex stage. You must compile OpenSSL 3.x and its FIPS provider from source for each target Android architecture.
1. Download OpenSSL Source: Download the source code for a version of OpenSSL 3.x that has a validated FIPS module (e.g., OpenSSL 3.1.2).
2. Set up Build Script: The following shell script automates the cross-compilation process. Save it as build_openssl_android.sh and make it executable (chmod +x build_openssl_android.sh).
#!/bin/bash
set -e

OPENSSL_VERSION="3.1.2" # Use the appropriate version
TOOLCHAIN="$ANDROID_NDK_HOME/toolchains/llvm/prebuilt/linux-x86_64"
OUTPUT_DIR="$(pwd)/openssl_build"

# Architectures to build for
ARCHS=( "android-arm64" "android-arm" "android-x86_64" "android-x86" )
ABIS=( "arm64-v8a" "armeabi-v7a" "x86_64" "x86" )

# Download and extract OpenSSL
if [ ! -f "openssl-${OPENSSL_VERSION}.tar.gz" ]; then
    wget "https://www.openssl.org/source/openssl-${OPENSSL_VERSION}.tar.gz"
fi
tar -xzf "openssl-${OPENSSL_VERSION}.tar.gz"

cd "openssl-${OPENSSL_VERSION}"

for i in "${!ARCHS[@]}"; do
    ARCH=${ARCHS[$i]}
    ABI=${ABIS[$i]}
    API=23 # Match minSdk

    echo "================================================="
    echo "Building OpenSSL for ${ABI}"
    echo "================================================="

    export PATH="$TOOLCHAIN/bin:$PATH"
    export ANDROID_API=${API}

    # Configure OpenSSL with FIPS provider support
    ./Configure ${ARCH} -D__ANDROID_API__=${API} enable-fips

    # Compile
    make -j$(nproc)

    # Install to our output directory
    make install_sw DESTDIR="${OUTPUT_DIR}/${ABI}"

    # Clean up for the next architecture
    make clean
done

echo "OpenSSL build complete. Libraries are in ${OUTPUT_DIR}"

3. Run the Build: Execute the script from the directory where you downloaded the OpenSSL source.
./build_openssl_android.sh

This process will take a significant amount of time. Upon completion, you will have a directory structure like openssl_build/arm64-v8a/usr/local/lib64/ containing the required .so files (libcrypto.so, libssl.so, and fips.so).
4.0 Stage 2: Source Code and Dependency Setup
1. Clone the Repository: Clone the forked Signal-Android repository. This guide assumes the libsignal fork is included as a git submodule.
git clone --recurse-submodules https://your-git-host.com/your-fork/Signal-Android.git
cd Signal-Android

2. Place Pre-compiled Dependencies:
? Create the directory structure required by our build scripts inside the fips-crypto-bridge module:
mkdir -p fips-crypto-bridge/libs/openssl/include

? Copy the OpenSSL header files:
cp -r ../openssl_build/arm64-v8a/usr/local/include/* fips-crypto-bridge/libs/openssl/include/

? For each architecture, copy the compiled shared object (.so) files into the correct ABI directory:
# For arm64-v8a
mkdir -p fips-crypto-bridge/libs/openssl/arm64-v8a
cp ../openssl_build/arm64-v8a/usr/local/lib64/libcrypto.so fips-crypto-bridge/libs/openssl/arm64-v8a/
cp ../openssl_build/arm64-v8a/usr/local/lib64/fips.so fips-crypto-bridge/libs/openssl/arm64-v8a/

# For armeabi-v7a
mkdir -p fips-crypto-bridge/libs/openssl/armeabi-v7a
cp ../openssl_build/armeabi-v7a/usr/local/lib/libcrypto.so fips-crypto-bridge/libs/openssl/armeabi-v7a/
cp ../openssl_build/armeabi-v7a/usr/local/lib/fips.so fips-crypto-bridge/libs/openssl/armeabi-v7a/

# ... repeat for x86_64 and x86 ...

This step is crucial, as it provides the artifacts that the CMakeLists.txt and build.gradle.kts scripts expect to find.
5.0 Stage 3: Building the Application
With the environment and dependencies in place, building the application is a single command.
1. Navigate to the Root Directory: Ensure you are in the root of the Signal-Android project directory.
2. Run the Gradle Wrapper: Execute the assembleDebug task. This will trigger the entire build chain:
? CMake compiles the C bridge.
? Cargo compiles the Rust bridge and links it against the C bridge and OpenSSL.
? Gradle compiles the Kotlin/Java code and packages everything into a debug APK.
./gradlew assembleDebug

3. Locate the APK: If the build is successful, the output APK will be located at:
app/build/outputs/apk/debug/app-debug.apk
You can now install this APK on an emulator or a physical device for testing.
6.0 Troubleshooting Common Issues
? NDK Path Not Found: Ensure ANDROID_NDK_HOME is set correctly and points to the exact version specified in the build scripts.
? Linker Errors (ld: cannot find -l...): This almost always means the .so files from Stage 2 were not placed in the correct directory, or you are missing an architecture. Double-check the paths in fips-crypto-bridge/libs/openssl/.
? CMake Errors: Ensure you have a compatible version of CMake installed and that the path to CMakeLists.txt in fips-crypto-bridge/build.gradle.kts is correct.
? Rust Compilation Errors: Ensure you have installed the correct Android targets for Rust as specified in the prerequisites.
